@nrk/core-css

This is the core CSS layer for NRK. It is not a framework that provides solutions for all use cases, but instead exposes a thin foundation.

Namespacing

@nrk/core-css namespaces all classes with .nrk-grid, to make it safer to include in your existing code base.

Grid and sizing

The grid consist of two major components: .nrk-grid (flexbox based with inline-block fallback), and 12 fluid size units. Units are prefixed with .nrk-xs-, .nrk-sm-, .nrk-md-, .nrk-lg- based on minimum widths, meaning they also apply to all sizes above unless other is specified (e.g., .nrk-xs-4 applies to extra small, small, medium and large devices). Rows wrap automatically. 

/* no media query */              /* Extra small devices */
@media (min-width: 640px) { ... } /* Small devices */
@media (min-width: 720px) { ... } /* Medium devices */
@media (min-width: 990px) { ... } /* Large devices */

Fill screen width

If you need to break out of a centered container and fill the entire screen width, you can use the class .nrk-xs-100. NB: Make sure there is a wrapper element in body with overflow:hidden to prevent .nrk-xs-100 from causing horizontal scrollbars.

Settings

SVGs will inherit the color from the closest parent. By setting the color on the wrapping element, the SVG will inherit that color property.

Accessibility

Accessibility considerations is a part of @nrk/core-css styling. Hide an element to all devices except screen readers with .nrk-sr, or make it show on focus by using .nrk-sr-focus. In general, semantic html-markup like <nav> does not require attributes for usability. Be aware though that providing aria- attributes can be useful when developing interactive widgets. Use .nrk-fade to achieve a more sublte look of elements (for instance meta data or icons in buttons).
